7. LESSONS LEARNED

7.1 Challenges Encountered and Solutions

Challenge 1: CORS Blocking Hosted Images
Issue: Initial Firebase Hosting deployment resulted in CORS errors blocking image requests from the web app.
Investigation: Browser DevTools showed "CORS error" and 404 responses for trophy image requests.
Solution: Added explicit CORS headers (Access-Control-Allow-Origin: *, Access-Control-Allow-Methods: GET, HEAD, OPTIONS) in firebase.json for /legends/**, /kits/**, /trophies/** paths.
Lesson: Cloud hosting configurations must explicitly enable cross-origin requests for web applications. Always test CORS behavior before production deployment.

Challenge 2: PowerShell Execution Policy Blocking Deployment
Issue: Initial Firebase deploy command blocked by PowerShell execution policy.
Investigation: PowerShell security settings prevented running the Firebase CLI script wrapper.
Solution: Used a direct CLI command invocation wrapper to bypass the execution policy restriction.
Lesson: Development environments may have security policies that affect deployment workflows. Document workarounds for team members.

Challenge 3: Image Filename Extension Mismatches
Issue: Some trophy images failed to load due to .png vs .jpg extension mismatches.
Investigation: Code referenced .png files while actual assets used .jpg extension.
Solution: Updated asset paths to match actual file extensions.
Lesson: File naming conventions and automated verification (e.g., asset path validation tests) prevent runtime failures.

Challenge 4: Managing Code Deprecations
Issue: Static analyzer reported 11 deprecation warnings for .withOpacity() usage.
Investigation: Flutter deprecated .withOpacity() in favor of .withValues() for precision.
Decision: Documented warnings for future remediation; did not address immediately due to time constraints.
Lesson: Deprecation warnings accumulate technical debt. Balance immediate fixes against project timeline constraints.
